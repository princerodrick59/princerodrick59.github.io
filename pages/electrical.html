Centralized Electrical Panel

This year we worked closely to create a singular electrical panel that has all core electronics mounted to it. This allows us to create a robust system that has most components in proximity to each other. We also decided to mount it to the back of our elevator so that we can access it without the use of tools.

CAN Bus Architecture

Last year we were able to implement a CANivore so we can have two different CAN networks. This lets us isolate our drivebase's communication from that of our manipulators — fixing the problem of high CAN bus utilization we had two years ago. Along with this, we have created a labeling system for CAN IDs across our different subsystems.

Dual Network Layout

Network 1Drivebase (CANivore) — 4× Drive Kraken x60, 4× Steer Kraken x44, Pigeon IMU
Network 2Manipulators (RIO CAN) — Intake, Spindexer, Indexer, Turret, Climber

Why Two Networks?

Isolation
Drivebase CAN traffic no longer competes with manipulator traffic. Swerve modules update faster and more reliably without high bus utilization degrading performance.
CAN ID Labeling
A consistent CAN ID labeling system is applied to all controllers, making it easy to identify devices at a glance during pit debugging and ensuring IDs never conflict between networks.
